# Governance

OllyGarden Skills is an OllyGarden-maintained open source project. It uses a maintainer-led
governance model: anyone may propose and contribute changes, while repository maintainers are
responsible for scope, quality, security, releases, and final merge decisions.

## Project principles

Decisions should keep the skills:

- focused on concrete OllyGarden workflows and opinions;
- layered on maintained upstream facts rather than duplicating them;
- token-efficient and designed for progressive disclosure; and
- conformant with the Agent Skills specification.

Vendor-neutral OpenTelemetry facts belong in the companion
[`ollygarden/opentelemetry-agent-skills`](https://github.com/ollygarden/opentelemetry-agent-skills)
repository.

## Decisions and changes

Routine changes are decided through public issue and pull request review. New skills, major scope
changes, and changes to shared conventions should begin with an issue so contributors and
maintainers can align before implementation. Maintainers seek practical consensus, but may make a
final decision when consensus is not possible, documenting the reasoning in the relevant issue or
pull request.

Security reports and conduct incidents are the exceptions to public discussion. Follow
[SECURITY.md](SECURITY.md) and [CODE_OF_CONDUCT.md](CODE_OF_CONDUCT.md) for private reporting.

## Roles

- **Contributors** report problems, propose improvements, author changes, review work, and help
  other users.
- **Maintainers** review and merge changes, manage releases and automation, enforce project scope
  and community policies, and respond to security and conduct reports.

Maintainer access is granted by the existing maintainers based on sustained, constructive
participation, sound judgment, and a demonstrated commitment to the project principles and Code of
Conduct. Access may be removed for inactivity, at the maintainer's request, or for conduct or
security reasons.
