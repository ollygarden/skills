# Security Policy

## Supported versions

This repository publishes agent instructions, scripts, and references directly from `main`. Only
the latest commit on `main` is actively maintained. Older revisions and downstream copies may no
longer reflect current security guidance.

## Reporting a vulnerability

Do not report suspected vulnerabilities in a public issue, discussion, or pull request. Email
[security@ollygarden.com](mailto:security@ollygarden.com) with:

- the affected skill, script, workflow, or other file;
- the impact and conditions needed to reproduce the issue;
- reproduction steps or a proof of concept, when available;
- any suggested remediation; and
- how you would like to be credited.

This includes vulnerabilities in executable scripts, GitHub Actions workflows, dependency handling,
and instructions that could cause an agent to perform an unsafe action. Ordinary factual or
documentation errors can be reported with the public bug-report form.

We will confirm receipt, investigate the report, coordinate remediation and disclosure with the
reporter, and credit the reporter unless they prefer to remain anonymous.
